INISIASI 1

Apa itu Struktur Data?

Definisi

Struktur data adalah cara mengorganisasikan dan menyimpan data di komputer agar bisa diakses dan dimodifikasi secara efisien. Struktur data ada di mana-mana dalam kehidupan digital kita.

Membuka File

Jutaan file diorganisir dalam struktur data agar bisa dicari cepat.

Cari Kontak

Hasil pencarian kontak dalam milidetik menggunakan algoritma SD.

Login Sosmed

Verifikasi akun terhadap jutaan pengguna menggunakan SD yang efisien.

Layanan Darurat

Lokasi penelepon langsung ditemukan via SD dan peta dikirim ke tim darurat.

Tiga kebutuhan efisiensi: Data set yang berkembang → Kecepatan proses → Jumlah operasi. Solusinya adalah organisasi data yang tepat menggunakan struktur data.
INISIASI 1

Interface & ADT

Interface (ADT)

Abstract Data Type — mendefinisikan operasi yang didukung beserta semantiknya. Contoh: Queue, Stack, List.

Implementasi

Kode nyata yang merealisasikan interface. Contoh: sorting diimplementasikan dalam Java menggunakan interface List.

Tiga Interface Utama
Queue — FIFO (First In, First Out)
add(x)/enqueue(x) masuk dari satu sisi, remove()/dequeue() keluar dari sisi lain. Contoh: antrian kasir.
Stack — LIFO (Last In, First Out)
push(x) menambah ke top, pop() mengambil dari top. Contoh: tumpukan piring, call stack.
Linear Sequence (List)
Diindeks 0 sampai n-1. Operasi: get(i), set(i,x), add(i,x), remove(i).

Doubly Linked List

Perbedaan DLL vs SLL

DLL punya dua pointer per node: prev (ke kiri) dan next (ke kanan). Traversal bisa dua arah.

AspekSLLDLL
Pointer/node1 (next)2 (prev+next)
Traversal→ satu arah↔ dua arah
Delete nodePerlu prev nodeLangsung O(1)
MemoriLebih hematLebih besar
Space-Efficient Linked List: setiap node menyimpan beberapa elemen (array kecil). Lebih cache-friendly, overhead pointer lebih sedikit. Trade-off: implementasi lebih kompleks.
